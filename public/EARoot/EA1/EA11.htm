<b>The Core CompassAE Engines At A Glance (“The Core Four”)</b><br /></p><p><br /></p><p>There are four foundational librariesin CompassAEon which the rest of the system is based. Each plugin is prepended with ‘erp_*’ so they are easy to locate.<br /></p><p><br /></p><p><b>erp_base_erp_svcs:</b> contains an implementation of the ubiquitous “party model” for managing people, organizations, roles, relationships and contact information. The models in this library are designed to be useful in a variety of circumstances, including headless SOA implementations and batch operations.<br /></p><p><br /></p><p>For this reason, the base library has very few dependencies. We opt instead to add incremental functionality to these models using ruby’s open classes and Rails’ plugin techniques. More on this later.<br /></p><p><br /></p><p><b>erp_tech_svcs:</b> this engine is implemented with the premise that services like logging, tracing and encryption would likely already exist in many organizations, so they are factored here so they can easily be re-implemented. There are default implementations here, and we track several excellent Rails projects as potential implementations of services like security and content/digital asset mgt.<br /></p><p><br /></p><p>The principle here is that it should be straightforward to extend or even swap out a default implementation of these services with existing implementations to suit a particular customer. In addition to this, CompassAE deployments should make it easy to:<br /></p><p><br /></p><p><ul><li>start with a Ruby-only or minimally native-dependent version of a service</li><br /></p><p><li>provide access to an open source, “enterprise” version of the same service</li><br /></p><p><li>allow for customers to plug into investments they have made in commercial products that implement the service</li><br /></p><p></ul><br /></p><p>An example of this is business rules processing. By default, CompassAE uses a pure Ruby implementation of the forward-chaining Rete algorithm called Ruleby. Compass also has a wrapper that allows implementers to plug CompassAE into the JBoss Drools rules processing engine, which is an open source, enterprise class rules processing engine. We provide an example Drools server in CompassAE called Axiom. This same wrapper API can be used by organizations that have investments in the IBM/iLog JRules or Fair Isaac Blaze commercial rules processing engines.<br /></p><p><br /></p><p><b>erp_dev_svcs:</b> this plugin exists to serve as a way to organize code that exists to support the development process, but will not live in the running production code.<br /></p><p><br /></p><p><b>erp_app:</b> provides an application infrastructure based on the Sencha/extjs UI framework, as well as several utilities and example applications. It houses the core application container framework and component model infrastructure that play a key role in the RAD/Agile orientation of CompassAE.<br /></p>
